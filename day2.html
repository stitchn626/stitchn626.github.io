This repository documents my journey of learning HTML, the foundation of building web pages. The goal is to understand HTML structure, tags, and best practices for creating clean, semantic web pages. This project serves both as a learning record and a practical portfolio of code examples and how to write reports on projects.

Introduction

HTML stands for HyperText Markup Language. It is used to define the structure and layout of a webpage. With HTML, we tell the browser what content to display and how it should be organized. While CSS styles the content and JavaScript adds behavior, HTML provides the skeleton.

Basic Structure Of An HTML Document

≤!DOCTYPE html ≥ ≤html≥ ≤head≥ ≤title≥ my first webpage ≤/title≥ ≤/head≥ ≤body≥ ≤h1≥ hello world ≤/h1≥ ≤ p≥ my first web page created. ≤/p≥ ≤/body≥ ≤/html≥

  • ≤!DOCTYPE html≥

    • This declaration defines the document as an HTML5 document. It must be the first line of your HTML file.

  • ≤html≥

    • The root element that wraps the entire content of your HTML page. All other elements go inside this tag.

  • ≤head≥

    • Contains metadata and information about the document that is not displayed directly on the web page. It can include:
    ≤title≥
    – sets the page title in the browser tab
    ≤meta≥
    – defines character set, viewport, etc.
    ≤link≥
    – connects to external stylesheets
    ≤title≥
    -Sets the title of the web page, which appears on the browser tab. Example: ≤title≥Home Page ≤/title≥

  • ≤body≥

    • Contains all the visible content of the web page, such as text, images, links, and more. Everything that appears in the browser is placed here.
